# Папка commands/cancel

Папка реализует единую команду `/cancel` и делегирование отмены по активным сценариям.

## Назначение

- Централизованно обрабатывать отмену в courier-flow и admin-flow.
- Возвращать пользователя в корректное состояние и интерфейс после отмены.
- Избежать дублирования логики `/cancel` внутри доменных модулей.

## Файлы

- `cancel.ts` — регистрация `/cancel`; оркестрация цепочки обработчиков отмены.
- `cancel-admin-auth-warehouse-flows.ts` — отмена auth/warehouse admin-шагов.
- `cancel-admin-edit-approval-flows.ts` — отмена сценариев редактирования админов и approve-flow.
- `cancel-admin-edit-approval-sim-flows.ts` — отмена SIM-сценариев.
- `cancel-admin-courier-flows.ts` — отмена сценариев редактирования курьеров и истории.

## Порядок обработки

1. `cancel.ts` проверяет текущий state.
2. Последовательно запускает специализированные `handleCancel*` обработчики.
3. Если ни один не сработал, выполняется общий fallback (очистка state/tempData и возврат клавиатуры).

## Правила изменения

1. Новый многошаговый state обязательно добавить в соответствующий `cancel-*` модуль.
2. Если сценарий возвращает в разные точки, фиксировать это на уровне конкретного flow-модуля, а не в `cancel.ts`.
3. При изменении state names синхронно обновлять проверки во всех `cancel-*` файлах.
