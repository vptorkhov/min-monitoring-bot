# Папка commands/return

Папка содержит сценарий сдачи устройства `/return_sim`.

## Файлы

- `return-sim.ts` — регистрация и выполнение потока сдачи устройства.
- `return-sim-parsers.ts` — парсинг пользовательских ответов (`Да/Нет`, тип повреждения).

## Что делает поток `/return_sim`

1. Проверяет регистрацию и активность курьера.
2. Проверяет наличие активной сессии.
3. Для личного устройства завершает сессию без шага повреждений.
4. Для обычного устройства запускает шаги:
   - вопрос о наличии повреждения;
   - выбор типа повреждения;
   - ввод комментария о повреждении (если нужно).
5. Завершает сессию и возвращает корректную курьерскую клавиатуру.

## UI и callback

- Поддерживаются и reply-кнопки, и inline-кнопки.
- Callback обрабатывается через общий callback-router, подключаемый при регистрации команды.

## Правила изменения

1. Любое изменение формулировок ответов синхронизировать с парсерами в `return-sim-parsers.ts`.
2. При добавлении новых шагов обновлять state transitions и fallback в `/cancel`.
