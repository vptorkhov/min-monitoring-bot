# Папка commands

Эта папка содержит обработчики команд Telegram-бота. Каждая команда реализована в отдельном файле.

## Файлы

### `index.ts`

Центральный файл регистрации всех команд бота. Выполняет:

- Импорт всех обработчиков команд
- Создание необходимых зависимостей (сервисы, репозитории)
- Регистрацию команд в боте

### `start.ts`

Обработчик команды `/start`. Выполняет:

- Проверку существования пользователя
- Запуск процесса регистрации для новых пользователей
- Приветствие зарегистрированных пользователей
- Показ подходящей reply-клавиатуры:
    - `🏠 Выбрать склад`, если склад не выбран
    - `🚲 Взять СИМ` / `🏠 Выбрать склад` / `❌🏠 Отвязаться от склада`, если склад выбран и нет активной сессии

### `set-warehouse.ts`

Обработчик команды `/set_warehouse`. Позволяет:

- Выбрать и прикрепиться к складу
- Показать список активных складов
- Проверить отсутствие активной сессии
- Выбрать склад тремя способами: ручной ввод номера, inline-кнопки `1..N`, reply-кнопки `1..N`
- Отменить шаг выбора склада через `❌ Отмена` (reply keyboard) или `/cancel`
- Запускаться не только из `/set_warehouse`, но и из reply-кнопки `🏠 Выбрать склад` (поддерживается также старый текст `🏠Выбрать склад`)
- После успешного выбора склада показывать основную клавиатуру активного курьера

### `clear-warehouse.ts`

Обработчик команды `/clear_warehouse`. Выполняет:

- Отвязку курьера от всех складов
- Проверку отсутствия активной сессии
- Запуск из `/clear_warehouse` и из reply-кнопки `❌🏠 Отвязаться от склада`

### `take-sim.ts`

Обработчик команды `/take_sim`. Реализует:

- Показ списка доступных СИМ на складе
- Создание новой сессии использования
- Приоритет личных устройств
- Запуск из `/take_sim` и из reply-кнопки `🚲 Взять СИМ`

### `return-sim.ts`

Обработчик команды `/return_sim`. Выполняет:

- Завершение активной сессии
- Опрос о повреждениях для не-личных устройств
- Обновление статуса устройства
- Возврат основной клавиатуры активного курьера после успешной сдачи СИМ (если снова выполнены условия показа)

### `cancel.ts`

Обработчик команды `/cancel`. Предоставляет:

- Сброс текущего состояния пользователя
- Очистку временных данных
- Возврат в исходное состояние

## Архитектура команд

Каждая команда:

1. Проверяет условия выполнения (регистрация, активная сессия и т.д.)
2. Выполняет основную логику
3. Отправляет ответ пользователю
4. Может устанавливать состояния для многошаговых процессов
