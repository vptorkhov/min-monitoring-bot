# Папка commands

Папка содержит регистрацию и реализацию команд Telegram-бота для двух режимов:

- courier-flow (пользовательские команды курьера);
- admin-flow (скрытый админский режим со своими сценариями).

## Структура

- `admin/` — auth-команды admin-mode и message-router для state-based шагов.
- `cancel/` — единая логика `/cancel` с делегированием по активным сценариям.
- `courier/` — команды редактирования курьеров в admin-flow.
- `return/` — сценарий `/return_sim` и парсинг ответов шага сдачи.
- `sim/` — команды управления SIM в admin-flow.
- `warehouse/` — команды выбора и управления складами в admin-flow.

Верхний уровень оставлен для публичных courier-команд и точки сборки команд.

## Верхнеуровневые файлы

- `index.ts` — точка регистрации всех команд; поднимает зависимости, callback-router и подключает модули.
- `start.ts` — обработчик `/start` и показ стартовой клавиатуры по состоянию курьера.
- `set-warehouse.ts` — обработчик `/set_warehouse` (включая запуск из кнопок и callback).
- `clear-warehouse.ts` — обработчик `/clear_warehouse`.
- `take-sim.ts` — обработчик `/take_sim`.
- `admin.ts` — оркестратор admin-mode: создает контексты и регистрирует доменные admin-команды.
- `admin.types.ts` — общие типы сессионных данных для admin-flow.

## Потоки

### Courier-flow

- Команды курьера работают через сервисы курьеров, складов и сессий.
- Reply/inline-кнопки нормализуются в команды.
- Общая отмена и возврат в корректную клавиатуру происходят через модуль `cancel/`.

### Admin-flow

- Вход через `/admin`.
- Команды в `admin/`, `sim/`, `warehouse/`, `courier/` управляют entry-point сценариями.
- Текстовые шаги многошаговых сценариев обрабатываются единым router в `admin/admin-message-router.ts`.

## Callback и state management

- Callback-обработчики регистрируются через единый `callback-router`, а не точечно в каждом модуле.
- Переходы состояний и временные данные хранятся в `stateManager`.
- Для admin-flow критично синхронно обновлять команды, states и cancel-handlers.

## Как расширять

1. Добавить новую команду в профильный доменный модуль.
2. При необходимости добавить message-state обработку в admin-router.
3. Добавить/обновить состояния в `states.constant.ts`.
4. Добавить отмену шага в `cancel/`.
5. Подключить регистрацию в `index.ts` или `admin.ts`.
