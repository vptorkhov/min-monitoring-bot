# Папка commands/courier

Папка содержит команды управления курьерами в admin-flow.

## Назначение

- Запуск сценариев выбора курьера (`admin` и `superadmin` уровни).
- Изменение параметров выбранного курьера (статус, ФИО).
- Просмотр истории сессий выбранного курьера.

## Файлы

- `courier-commands.ts` — доменный оркестратор; подключает selection/action/history команды.
- `courier-selection-commands.ts` — команды выбора курьера (`/admin_edit_couriers`, `/superadmin_edit_couriers`).
- `courier-action-commands.ts` — команды изменения данных курьера (статус, ФИО).
- `courier-history-commands.ts` — команды истории курьера.

## Зависимости

- `AdminService` для проверок прав и доступа к сценариям.
- `SessionRepository` для истории сессий курьера.
- Контекстные callbacks из `commands/admin/*-message-context.ts` для резолва выбранного курьера и вывода карточек.

## Правила изменения

1. Для новой команды по выбранному курьеру использовать общий резолвер выбранного курьера.
2. Проверки прав и доступности команды выполнять до смены state.
3. Все новые state-based шаги должны быть продублированы в cancel-flow и message-handlers.
