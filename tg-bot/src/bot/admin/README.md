# Папка admin

Эта папка содержит служебную логику админского режима Telegram-бота.

## Файлы

### `admin-mode.ts`

Утилиты переключения и контроля админского режима:

- `enterAdminMode(telegramId)` — очищает текущий пользовательский сценарий и включает админ-режим
- `exitAdminMode(telegramId)` — выключает админ-режим
- `isUserInAdminMode(telegramId)` — проверка текущего админского режима
- `blockIfAdminGuestCommandNotAllowed(...)` — guard, который блокирует курьерские команды, если пользователь в админском режиме без авторизации

### `admin-command-hints.ts`

Единый источник текстов подсказок для авторизованного админ-режима:

- `getAdminCommandListMessage(adminPermissionsLevel, isWarehouseSelected)` — формирует сообщение `Список команд:` для 4 вариантов (админ/суперадмин × склад выбран/не выбран)
- `isAuthenticatedAdminState(state)` — утилита проверки, что состояние является `admin_authenticated` или `admin_authenticated_with_warehouse`

## Текущие ограничения

В неавторизованном админском режиме разрешены только:

- `/admin`
- `/admin_login`
- `/admin_register`
- `/admin_logout`
- `/exit_admin`
- `/cancel`

Реализованные сценарии:

- `/admin_register` — пошаговая регистрация админа (логин -> пароль -> создание неактивной записи)
- `/admin_login` — пошаговый вход админа (логин -> пароль -> переход в `admin_authenticated` или `admin_authenticated_with_warehouse`, если у админа уже выбран склад в БД); после успешного входа бот отправляет сообщение `Список команд:` в соответствии с ролью (админ/суперадмин) и фактом выбора склада
- `/admin_change_password` — смена пароля для авторизованного админа/суперадмина; шаг: ввод нового пароля (мин. 6 символов) -> обновление пароля в БД -> возврат в авторизованное состояние без разлогина
- `/admin_set_warehouse` — выбор склада для админа; доступно любому авторизованному админу; шаги: показать список активных складов → ввод номера склада → сохранение `admins.warehouse_id`; при невалидном номере бот продолжает ожидать корректный ввод; команда доступна как при невыбранном, так и при уже выбранном складе (для смены склада)
- `/admin_clear_warehouse` — отвязка админа от склада; доступно только если склад уже выбран (`admins.warehouse_id IS NOT NULL`); после успеха бот очищает связь и возвращает админа в состояние без выбранного склада
- `/admin_add_sim` — добавление нового СИМ; доступно любому авторизованному админу при выбранном складе; шаги: ввод номера СИМ (ровно 6 символов: 3 буквы латиница или кириллица + 3 цифры в произвольном порядке, например `АА000А`) → проверка дубликата по всей таблице → создание записи в `mobility_devices` с `is_personal=false`, `status='ok'`, `is_active=true`, привязанной к складу; номер сохраняется в верхнем регистре; при некорректном формате или дубликате бот продолжает ожидать корректный ввод; после успеха возврат в `admin_authenticated_with_warehouse`
- `/admin_active_sessions` — просмотр активных сессий выбранного склада; доступно авторизованному админу и суперадмину только при выбранном складе; выводит список в формате `*ФИО курьера* - *СИМ*` (для личного устройства выводится `Личный`); если активных сессий нет, выводится информативное сообщение; после выполнения команды админ возвращается в состояние, из которого команда была вызвана
- `/admin_sessions_history` — просмотр истории сессий выбранного склада за дату; доступно авторизованному админу и суперадмину только при выбранном складе; шаги: ввод даты в формате `ДД.ММ.ГГГГ` → валидация формата/существования даты → вывод списка сессий выбранного склада, у которых `start_date` попадает в выбранный московский день (включая активные и завершенные), от новейшей к старейшей; формат элемента: ФИО, СИМ (`Личный` для персонального), дата начала, дата конца (если есть), статус СИМ после сессии, комментарий состояния (если есть); после выполнения команды возврат в состояние до вызова
- `/admin_sim_interactions` — взаимодействие с конкретными СИМ выбранного склада; доступно любому авторизованному админу при выбранном складе; шаги: показать нумерованный список СИМ (без личного транспорта) → выбор СИМ по номеру в списке или по номеру СИМ → показ карточки СИМ (номер, статус активности, статус исправности, последнее сообщение о неисправности для `warning`/`broken`, наличие активной сессии и ФИО курьера) и доступных команд:
	- `/admin_sim_change_active` — изменить `is_active` (`1`/`1.`/`Активный`/`1. Активный` или `2`/`2.`/`Отключен`/`2. Отключен`), команда недоступна при активной сессии по выбранному СИМ; после успеха возврат к карточке выбранного СИМ
	- `/admin_sim_change_status` — изменить `status` (`1`/`1.`/`Исправен`/`1. Исправен`, `2`/`2.`/`Поврежден`/`2. Поврежден`, `3`/`3.`/`Сломан`/`3. Сломан`), команда недоступна при активной сессии по выбранному СИМ; при выборе `Сломан` дополнительно устанавливается `is_active=false`; после успеха возврат к карточке выбранного СИМ
	- `/admin_sim_story` — показать историю сессий выбранного СИМ (время в формате ЧЧ:ММ по Москве, ФИО курьера, состояние после сессии, комментарий), после показа бот автоматически возвращает к выбору команды по этому СИМ
	- `/admin_sim_delete` — удалить выбранный СИМ после подтверждения `ДА`; если по СИМ есть активная сессия, удаление запрещено; если есть только завершенные сессии, СИМ удаляется вместе с его историей в `session`
- `/admin_logout` — выход из авторизованного админского состояния в `admin_guest_mode`
- `/exit_admin` — полный выход из админского режима в курьерский поток
- `/superadmin_create_warehouse` — создание нового склада; доступно только суперадмину (`permissions_level >= 2`); шаги: название (мин. 2 символа) → адрес (мин. 2 символа) → создание записи в БД; доступна как с выбранным складом, так и без него
- `/superadmin_edit_warehouses` — редактирование существующих складов; доступно только суперадмину (`permissions_level >= 2`); шаги: выбор номера склада из списка → выбор действия над складом
	- `/superadmin_edit_warehouse_name` — изменить название (мин. 2 символа), после успеха возврат к выбору действия по складу
	- `/superadmin_edit_warehouse_address` — изменить адрес (мин. 2 символа), после успеха возврат к выбору действия по складу
	- `/superadmin_edit_warehouse_status` — изменить статус (`1`/`1.`/`Активный`/`1. Активный` или `2`/`2.`/`Отключен`/`2. Отключен`), после успеха возврат к выбору действия по складу
	- `/superadmin_edit_warehouse_delete` — удалить склад после подтверждения `ДА`; после удаления возврат в состояние до запуска `/superadmin_edit_warehouses`
- `/superadmin_edit_admins` — редактирование существующих администраторов; доступно только суперадмину (`permissions_level >= 2`); шаги: выбор номера администратора из списка (кроме суперадминов) → выбор действия над администратором
	- `/superadmin_edit_admin_status` — изменить статус (`1`/`1.`/`Активный`/`1. Активный` или `2`/`2.`/`Отключен`/`2. Отключен`), после успеха возврат к выбору действия по администратору
	- `/superadmin_edit_admin_delete` — удалить администратора после подтверждения `ДА`; после удаления возврат к обновленному списку администраторов
	- `/superadmin_edit_admin_password` — изменить пароль (мин. 6 символов), после успеха возврат к выбору действия по администратору
- `/admin_apply_registrations` — принятие регистраций курьеров; доступно авторизованному админу и суперадмину (с выбранным складом и без него); шаги: показать нумерованный список неактивных курьеров без записей в `session` → ввод номера курьера → подтверждение `Да`/`Нет`
	- при ответе `Да` (в любом регистре) бот активирует курьера (`is_active=true`) и возвращает к обновленному списку неактивных
	- при ответе `Нет` (в любом регистре) бот сообщает, что курьер остался неактивным, и возвращает к обновленному списку неактивных
	- при ином ответе бот выводит ошибку и повторно ожидает корректный ответ
- `/admin_edit_couriers` — взаимодействие с курьерами выбранного склада; доступно авторизованному админу и суперадмину **только при выбранном складе** (`admins.warehouse_id IS NOT NULL`); шаги: показать нумерованный список (все курьеры с `warehouse_id` = выбранный склад) → ввод номера курьера → показ карточки курьера (ФИО, телефон, никнейм, склад, статус активности, инфо об активной сессии если есть) и доступных команд:
	- `/admin_edit_courier_status` — изменить `is_active` (`1`/`1.`/`Активный`/`1. Активный` или `2`/`2.`/`Отключен`/`2. Отключен`); после успеха возврат к обновлённой карточке курьера
	- `/admin_courier_history` — показать последние 50 сессий курьера в формате `*номер СИМ* начало:*ДД.ММ.ГГГГ ЧЧ:ММ* конец:*ДД.ММ.ГГГГ ЧЧ:ММ*` (конец только если сессия завершена); после показа ожидается ввод `ДА` (строго верхний регистр) для полной истории или `/cancel` / `нет` (любой регистр) для возврата к карточке
- `/superadmin_edit_couriers` — взаимодействие со **всеми** курьерами; доступно только суперадмину (`permissions_level >= 2`); доступно как при выбранном складе, так и без него; показывает всех курьеров, в том числе не выбравших склад; подкоманды:
	- `/superadmin_edit_courier_status` — аналог `/admin_edit_courier_status`
	- `/superadmin_courier_history` — аналог `/admin_courier_history`
	- все подкоманды (`*_edit_courier_status`, `*_courier_history`) доступны только после запуска базовой команды (`/admin_edit_couriers` или `/superadmin_edit_couriers`) и выбора конкретного курьера

Поведение `/cancel` в редактировании складов:

- на этапе выбора склада или выбора действия — возврат в состояние до запуска `/superadmin_edit_warehouses`
- на этапах ввода нового названия/адреса/статуса/подтверждения удаления — возврат к выбору действия по выбранному складу
- ввод подкоманд `/superadmin_edit_warehouse_*` без активного контекста выбора склада приводит к сообщению о недоступности команды

Общее правило показа подсказки `Список команд:`:

- если `/cancel` возвращает пользователя в `admin_authenticated` или `admin_authenticated_with_warehouse`, после сообщения об отмене бот дополнительно отправляет актуальное сообщение `Список команд:` для текущей роли и состояния склада
- если выполнение админской команды завершилось возвратом в `admin_authenticated` или `admin_authenticated_with_warehouse`, бот также повторно показывает `Список команд:` (например, когда список сессий пуст, список СИМ пуст, список курьеров пуст и т.д.)
- если `/cancel` вызван в «idle»-состоянии (`admin_guest_mode`, `admin_authenticated`, `admin_authenticated_with_warehouse`) без активного подрежима, бот отвечает только `ℹ️ Нет активного действия для отмены.`

Поведение `/cancel` в редактировании администраторов:

- на этапе выбора администратора — возврат в состояние до запуска `/superadmin_edit_admins`
- на этапе выбора действия по администратору — возврат к выбору администратора с повторным выводом списка
- на этапах ввода нового статуса/подтверждения удаления/нового пароля — возврат к выбору действия по выбранному администратору
- ввод подкоманд `/superadmin_edit_admin_*` без активного контекста выбора администратора приводит к сообщению о недоступности команды

Поведение `/cancel` в принятии регистраций курьеров:

- на этапе выбора курьера (`/admin_apply_registrations`) — возврат в состояние, из которого была вызвана команда
- на этапе подтверждения `Да`/`Нет` — возврат к обновленному списку неактивных курьеров
- если на момент возврата список кандидатов пуст — возврат в состояние до запуска `/admin_apply_registrations`

Поведение `/cancel` во взаимодействии с курьерами (`/admin_edit_couriers` и `/superadmin_edit_couriers`):

- на этапе выбора курьера — возврат в состояние до запуска команды
- на этапе выбора действия по курьеру — возврат к списку курьеров
- на этапах ввода нового статуса или ожидания полной истории — возврат к карточке выбранного курьера; ввод `нет` (в любом регистре) также возвращает к карточке на этапе ожидания полной истории

Поведение `/cancel` в смене пароля:

- на этапе ввода нового пароля (`/admin_change_password`) — возврат в `admin_authenticated` без разлогина

Поведение `/cancel` в выборе склада админа:

- на этапе ввода номера склада (`/admin_set_warehouse`) — возврат в состояние, из которого была запущена команда

Поведение `/cancel` в добавлении СИМ:

- на этапе ввода номера СИМ (`/admin_add_sim`) — возврат в `admin_authenticated_with_warehouse`

Поведение `/cancel` в просмотре истории сессий:

- на этапе ввода даты (`/admin_sessions_history`) — возврат в состояние, из которого была вызвана команда

Поведение `/cancel` во взаимодействии с СИМ:

- на этапе выбора СИМ (`/admin_sim_interactions`) бот дополнительно показывает подсказку `/cancel - вернуться в состояние выбранного склада`; при вводе `/cancel` — возврат в `admin_authenticated_with_warehouse`
- на этапах выбора действия, изменения активности, изменения исправности, подтверждения удаления — возврат к обновленному списку СИМ
- если список СИМ после удаления/обновления пуст, возврат в `admin_authenticated_with_warehouse`

## Состояния admin_mode_states

Все состояния, при которых `isUserInAdminMode()` возвращает `true`:

- `admin_guest_mode` — неавторизованный режим
- `admin_register_awaiting_login` / `admin_register_awaiting_password` — шаги регистрации
- `admin_login_awaiting_login` / `admin_login_awaiting_password` — шаги входа
- `admin_authenticated` — авторизованный режим
- `admin_authenticated_with_warehouse` — авторизованный режим с выбранным складом
- `admin_change_password_awaiting_new` — ожидание нового пароля в сценарии `/admin_change_password`
- `admin_set_warehouse_selecting` — ожидание номера склада в сценарии `/admin_set_warehouse`
- `admin_create_warehouse_awaiting_name` / `admin_create_warehouse_awaiting_address` — шаги создания склада
- `admin_edit_warehouses_selecting` — ожидание номера склада для редактирования
- `admin_edit_warehouse_action_selecting` — ожидание выбора действия по выбранному складу
- `admin_edit_warehouse_awaiting_name` — ожидание нового названия склада
- `admin_edit_warehouse_awaiting_address` — ожидание нового адреса склада
- `admin_edit_warehouse_awaiting_status` — ожидание нового статуса склада
- `admin_edit_warehouse_awaiting_delete_confirm` — ожидание подтверждения удаления склада (`ДА`)
- `admin_edit_admins_selecting` — ожидание номера администратора для редактирования
- `admin_edit_admin_action_selecting` — ожидание выбора действия по выбранному администратору
- `admin_edit_admin_awaiting_status` — ожидание нового статуса администратора
- `admin_edit_admin_awaiting_delete_confirm` — ожидание подтверждения удаления администратора (`ДА`)
- `admin_edit_admin_awaiting_password` — ожидание нового пароля администратора
- `admin_apply_registrations_selecting` — ожидание номера курьера в сценарии `/admin_apply_registrations`
- `admin_apply_registration_awaiting_confirm` — ожидание ответа `Да`/`Нет` для подтверждения принятия регистрации
- `admin_sessions_history_awaiting_date` — ожидание даты `ДД.ММ.ГГГГ` в сценарии `/admin_sessions_history`
- `admin_add_sim_awaiting_number` — ожидание номера нового СИМ в сценарии `/admin_add_sim`
- `admin_sim_interactions_selecting` — ожидание выбора СИМ в сценарии `/admin_sim_interactions`
- `admin_sim_interaction_action_selecting` — ожидание выбора действия для выбранного СИМ
- `admin_sim_interaction_awaiting_active_status` — ожидание нового статуса активности СИМ
- `admin_sim_interaction_awaiting_condition_status` — ожидание нового статуса исправности СИМ
- `admin_sim_interaction_awaiting_delete_confirm` — ожидание подтверждения удаления СИМ (`ДА`)
- `admin_edit_couriers_selecting` — ожидание номера курьера в сценарии `/admin_edit_couriers`
- `admin_edit_courier_action_selecting` — ожидание выбора действия для выбранного курьера
- `admin_edit_courier_awaiting_status` — ожидание нового статуса активности курьера
- `admin_courier_history_awaiting_full` — ожидание ответа на предложение показать полную историю курьера
- `superadmin_edit_couriers_selecting` — ожидание номера курьера в сценарии `/superadmin_edit_couriers`
- `superadmin_edit_courier_action_selecting` — ожидание выбора действия для выбранного курьера (суперадмин)
- `superadmin_edit_courier_awaiting_status` — ожидание нового статуса активности курьера (суперадмин)
- `superadmin_courier_history_awaiting_full` — ожидание ответа на предложение показать полную историю (суперадмин)
