# Папка admin

Папка содержит инфраструктуру admin-mode: хранение режима, guard-проверки и генерацию подсказок по доступным admin-командам.

## Состав

### admin-mode.ts

Центральный модуль состояния админского режима.

- Определяет набор состояний, которые считаются админскими.
- Включает режим через `enterAdminMode` с очисткой пользовательского сценария.
- Выключает режим через `exitAdminMode`.
- Проверяет режим через `isUserInAdminMode`.
- Блокирует нерелевантные команды в guest-режиме админа через `blockIfAdminGuestCommandNotAllowed`.

В guest-режиме админа разрешены только:

- `/admin`
- `/admin_login`
- `/admin_register`
- `/admin_logout`
- `/exit_admin`
- `/cancel`

### admin-command-hints.ts

Модуль текстовых подсказок для авторизованного админа.

- `getAdminCommandListMessage(adminPermissionsLevel, isWarehouseSelected)` формирует актуальный список команд по роли и контексту склада.
- `isAuthenticatedAdminState(state)` проверяет, что состояние относится к авторизованным (`AUTHENTICATED`, `AUTHENTICATED_WITH_WAREHOUSE`).

## Границы ответственности

- Эта папка не реализует пошаговые admin-сценарии ввода/редактирования.
- Все многошаговые сценарии находятся в командах и обработчиках в папке `src/bot/commands`.
- Папка `admin` предоставляет только базовые примитивы режима и подсказки.

## Как расширять

- Добавляйте новый guard или проверку режима только в `admin-mode.ts`.
- Не дублируйте списки разрешенных guest-команд в других местах.
- Любое изменение набора авторизованных состояний должно быть синхронизировано с `AdminState`.
