# Папка services

Эта папка содержит бизнес-логику приложения — сервисы, которые координируют работу репозиториев и реализуют сложные операции.

## Файлы

### `courier.service.ts`

Сервис для работы с курьерами. Реализует:

- **Регистрацию**: валидация и создание учетных записей
- **Проверку существования**: поиск по Telegram ID
- **Управление складами**: прикрепление/открепление
- **Активацию**: управление статусом учетной записи

### `warehouse.service.ts`

Сервис для работы со складами. Предоставляет:

- **Получение активных складов**: список доступных складов
- **Валидацию**: проверка существования склада

### `session.service.ts`

Сервис для управления сессиями использования СИМ. Реализует:

- **Создание сессий**: начало использования устройства с записью в БД
- **Завершение сессий**: сдача устройства с сохранением статуса СИМ и комментария о повреждениях
- **Проверки**: проверка активных сессий, конфликты одновременного использования
- **Получение информации**: получение текущей активной сессии, форматирование названия устройства

### `activation-notifier.service.ts`

Фоновый сервис уведомлений. Выполняет:

- **Мониторинг активации**: отслеживание новых активных курьеров
- **Отправка уведомлений**: автоматические сообщения в Telegram
- **Отслеживание отправки**: сохранение времени отправки в БД для предотвращения дублирования уведомлений при перезапусках

**Как это работает:**

- Периодически проверяет курьеров с `is_active = true` и `notified_at = NULL`
- При отправке уведомления сохраняет время отправки в поле `notified_at`
- После перезапуска бота уведомления отправляются только новым активированным курьерам

## Архитектура сервисов

Сервисы:

- Инкапсулируют бизнес-логику
- Координируют несколько репозиториев
- Предоставляют высокоуровневый API
- Обрабатывают ошибки и валидацию
- Возвращают типизированные результаты

## Отличие от репозиториев

- **Репозитории**: прямой доступ к данным, SQL-запросы
- **Сервисы**: бизнес-логика, валидация, координация
