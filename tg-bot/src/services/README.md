# Папка services

Эта папка содержит бизнес-логику приложения: сервисы координируют работу репозиториев и реализуют прикладные сценарии.

## Файлы

### `courier.service.ts`

Сервис для работы с курьерами. Реализует:

- Регистрацию: валидация и создание учетных записей
- Проверку существования: поиск по Telegram ID
- Управление складами: прикрепление и отвязка
- Активацию: проверка статуса учетной записи

### `warehouse.service.ts`

Сервис для работы со складами. Предоставляет:

- Получение активных складов
- Валидацию выбора склада

### `session.service.ts`

Сервис для управления сессиями использования СИМ. Реализует:

- Создание сессий
- Завершение сессий с сохранением статуса СИМ и комментария о повреждениях
- Проверки активных сессий
- Получение текущей активной сессии и форматирование названия устройства

### `activation-notifier.service.ts`

Фоновый сервис уведомлений. Выполняет:

- Мониторинг новых активированных курьеров
- Отправку уведомления в Telegram
- Отправку inline-кнопки и reply-кнопки `🏠 Выбрать склад`
- Отметку времени отправки в `notified_at`, чтобы не дублировать уведомления после перезапуска

## Архитектура сервисов

Сервисы:

- Инкапсулируют бизнес-логику
- Координируют несколько репозиториев
- Предоставляют высокоуровневый API
- Возвращают типизированные результаты

## Отличие от репозиториев

- Репозитории: прямой доступ к данным, SQL-запросы
- Сервисы: бизнес-логика, валидация, координация

## Пример использования

```typescript
const notifier = new ActivationNotifier(bot, courierService);
notifier.start(); // запуск мониторинга
notifier.stop(); // остановка мониторинга
```
